# 3. 学生視点エージェント（学びと成長）

## 【主な質問・懸念】

### 3.1 参加動機の明確化：「なぜこのプロジェクトに参加したいのか」
**質問：学生が「自分たちのやりたい」と感じられる要素は何か？**

**学生が参加を決める際に重要な心理要因：**

1. **「社会課題解決」への志向**
   - 「難病への偏見」という課題が「自分も解決したい」と思えるのか？
   - それとも「学校の課題だから」という義務感なのか？

2. **「キャリア形成」への期待**
   - 「企画提案」という経験が「就活で役立つ」と思えるのか？
   - 「実社会での経験」として価値を感じるのか？

3. **「人間関係」の構築**
   - 一緒に活動する仲間との「絆」を深めたいと思うのか？
   - 「新しい人間関係」を形成したいと思うのか？

**現在の企画案の問題点：**

```
企画案で提示されているもの：
「8月イベント参加 → 課題発見 → 企画提案 → 実施」

学生目線での「魅力度」評価：
- 社会課題解決？：「難病」という遠い課題。自分ごとになるのか？
- キャリア形成？：「企画提案」という単語だけでは、何ができるか不明
- 人間関係？：「企画提案」を「個人か、グループか」が不明
- 達成感？：「成功する見込み」が見えない。失敗もあり得るのか？
```

**学生からの疑問（予想）：**
- 「このプロジェクトに参加すると、具体的に何が変わるのか？」
- 「9月の企画提案で『却下』されたらどうなるのか？」
- 「他の学生も参加するのか、それとも自分たちだけなのか？」
- 「単位をもらえるのか？」
- 「実装後、何か成果が出たら、自分たちに報告が来るのか？」

---

### 3.2 グループダイナミクスと仲間との関係性
**質問：グループで取り組むことで、仲間との関係性は深まるか？**

**想定される学生の心理：**

1. **「プロジェクト型学習」への期待と懸念**
   - 期待：「友人と一緒に『本気の仕事』ができる」という特別感
   - 懸念：「意見の対立」「役割分担の不公平感」「グループ内の孤立」

2. **「リーダーシップ」の問題**
   ```
   学生グループの中で：
   - 誰が「企画のリーダー」になるのか？
   - 決定権は「全員一致」か、「少数決」か、「リーダーの専権」か？
   - リーダーに選ばれなかった学生の「やる気」は維持されるのか？
   ```

3. **「失敗時の心理的負担」**
   - 企画が「却下される」と、グループの雰囲気が悪くなるのでは？
   - 「あいつのせいで却下された」という相互非難が起きるのでは？
   - 大学が「グループ内の対立」にどう対応するのか？

**ピアラーニングの可能性と現在の不明確性：**

| 段階 | ピアラーニング | 現企画での位置づけ |
|------|---|---|
| 課題分析 | 複数の視点から議論 | 記述されていない |
| 企画作成 | 各自の経験・知識を共有 | 「グループ作成」と書かれているだけ |
| 実施 | 役割分担と相互支援 | 記述されていない |
| 振り返り | 失敗からの学習を共有 | 記述されていない |

**学生からの懸念（予想）：**
- 「課題分析の時点で『意見が分かれた』ら、どう調整するのか？」
- 「企画案を『共有すること』で、個人の創意性が損なわれるのでは？」
- 「実施時に『うまくいかない』と、グループ内で責任押し付けが起きるのでは？」

### 3.3 キャリア形成における価値
**質問：「企画を提案する」という経験は、キャリア形成に役立つか？**

**学生のキャリア視点での評価基準：**

1. **「就活での評価」**
   - ESや面接で「このプロジェクト」をどう説明するのか？
   - 「企画提案」という実績が「採用企業に価値があるのか」？
   - 企業が「何を評価するのか」は明記されているのか？

2. **「将来の仕事に必要なスキル」の習得**
   ```
   企画提案を通じて習得できるスキル：
   - 課題分析力：社会課題を複数の視点から理解
   - 提案力：ステークホルダーを説得する資料・プレゼン
   - 実装力：計画通りに実行する能力
   - 対人スキル：複数のステークホルダーとの折衝
   - メタ認知：「自分たちの判断は正しかったか」を問い直す力

   → これらが「現企画で明示的に教えられるのか」？
   ```

3. **「業界理解」の深化**
   - 難病患者支援の現場を経験できるのか？
   - 企業の「社会貢献活動」の実態を学べるのか？
   - NPOやコーディネーターの仕事を理解できるのか？

**学生からの問い（予想）：**
- 「このプロジェクトで『何ができるようになる』のか、教えてください」
- 「卒業後、このスキルは『どんな業界・職種』で役立つのか？」
- 「就活で『このプロジェクトの成果』をどう評価してもらえるのか？」

---

### 3.4 自発性と「やらされている感覚」
**質問：学生が「やりたいこと」として参加できるか、それとも「義務」か？**

**学生の「納得感」を判定する指標：**

1. **「参加の自由度」**
   ```
   現状の問い：
   - 参加は「完全に任意」なのか？
   - 「ゼミの一環」として「半強制」なのか？
   - 「成績評価に関わる」のか？
   - 「単位認定」されるのか？

   → これらで「学生の納得度」が大きく変わる
   ```

2. **「テーマ選択の自由度」**
   ```
   現状：「難病への偏見・情報発信」という課題が固定

   学生からの疑問：
   - 「この課題が嫌だったら、別のテーマでもいいのか？」
   - 「複数の課題から『自分たちで選ぶ』ことができるのか？」
   - 「新しい課題を『自分たちが提案』することはできるのか？」

   → テーマ選択の自由度がなければ、「やらされている感」は避けられない
   ```

3. **「決定権の実感」**
   ```
   現状：学生 → 企業・津田先生による「承認」

   学生からの疑問：
   - 「企業に却下されたら、何もできないのか？」
   - 「企業の意見に反対だったら、『No』と言えるのか？」
   - 「実施中に『計画変更』したいときは、誰の許可が必要か？」

   → 決定権がなければ、学生は「実行部隊」に過ぎない
   ```

**「やらされている感」を生む要因：**
- テーマが固定されている
- 企画内容の最終承認を「他者」に委ねている
- 「失敗するかもしれない」という不安が解消されていない
- 「成功時の喜び」と「失敗時のフォロー」の見通しが不明

---

### 3.5 成功体験と失敗からの学習
**質問：成功の達成感、失敗からの学習経験は用意されているか？**

**学生の「学習心理」の観点：**

1. **「成功体験」の必要性**
   ```
   学生が「このプロジェクトで学べた」と感じるには：

   ✓ 小さな成功を積み重ねる：課題分析で「複数の視点を発見できた」など
   ✓ 中間での達成感：企画提案が「承認された」瞬間
   ✓ 最終的な成果：実装後に「社会への影響」を実感する
   ✓ 自分の成長を「他者の評価」で認識：津田先生の面談など

   → 現企画では、これらの「成功体験の設計」が見えない
   ```

2. **「失敗からの学習」**
   ```
   もし「企画案が却下」されたら：

   最悪シナリオ：
   - 学生が「自分たちの判断は間違っていた」と落ち込む
   - グループ内で「誰のせいで落ちたのか」と責任転嫁
   - その後の「モチベーション」が完全に失われる

   → 大学がこれに「対応する仕組み」を用意しているのか？
   ```

3. **「振り返りと内省」の機会**
   ```
   学生が「何を学んだか」を認識するには：

   必要な機会：
   - 「中間振り返り」：8月イベント参加後、「課題の複雑さ」を言語化
   - 「企画提案での気づき」：提案作成で「実現可能性の制約」を理解
   - 「実施後の振り返り」：成果と課題を分析

   → これらが「誰によって」「どのような形で」実施されるのか？
   ```

**学生からの問い（予想）：**
- 「もし企画が『却下』されたら、その後どうなるのか？」
- 「実施後に『期待以下の成果』だった場合、それをどう評価するのか？」
- 「プロジェクト終了後、自分たちが『何を学んだか』を認識する時間はあるのか？」

## 【改善提案】

### 提案1：参加動機を高める「プロジェクト紹介」の工夫
```
【修正案】学生にとって「魅力的」にプレゼンする内容

【現在の説明】
「8月のイベントに参加して、課題を発見し、企画提案をします。
承認されれば、その企画を実施します。」

【修正案：「3つの価値」を明示】

価値1：「社会への直接的な貢献」
- 「あなたたちの企画が、実際に『難病患者の支援』につながります」
- 「単なるレポート提出ではなく、『社会で実装される』経験です」
- 「この経験は、『社会人になっても役立つ』最高の教材です」

価値2：「就活・キャリア形成での強み」
- 「『企画の立案から実施』まで一貫して経験できます」
- 「面接で『実際にやったプロジェクト』として説明できます」
- 「企業の人事担当者は『こういう経験をした学生』を高く評価します」

価値3：「仲間との絆と成長」
- 「『本気の課題』に一緒に取り組むことで、仲間との関係が深まります」
- 「困難を乗り越える過程で、『自分たちの可能性』を実感できます」
- 「卒業後も『このメンバーと一緒にやったプロジェクト』は、大切な思い出になります」

【具体的なシナリオ提示】
「もし、あなたたちの企画が『社内啓発キャンペーン』だったとしたら：
1. 8月：イベントで実際に難病患者の声を聞く
2. 9月：『どうやって啓発するか』をあなたたちが企画する
3. 10月～11月：その企画を星野社長の会社の中で実装
4. 12月：実装後、『社員の行動がどう変わったか』をデータで確認
5. その後：『あなたたちの企画で、社員の意識が変わった』という『成果』が残る

これは『就活でも話題になる』レベルの経験です。」

【募集時の資料】
- プロジェクト全体のビジュアル（ガントチャート）
- 「参加した学生の声」（過去のプロジェクトがあれば）
- 「企業側の期待」（星野社長からのメッセージ）
- 「津田先生の関与」（学生の安全・学習を担保する情報）
```

**効果：**
- 学生の「参加動機」が明確になる
- 「義務感」ではなく「自発的な選択」になる
- 「中途で脱落する学生」が減る

### 提案2：グループダイナミクスを支えるファシリテーション
```
【修正案】学生グループの「成長」を設計する

【グループ構成】
- コアメンバー：5名（全行程に参加）
- サポート：3名（イベント参加のみまたは企画提案のみ）
- オブザーバー：1～2名（最後の振り返り段階に参加）
→ 「全員参加」を強制せず、段階的な関与を認める

【グループ内の役割分担】
事前に津田先生が「役割分担ワークショップ」を実施

例：
- リーダー（1名）：全体の進捗管理
- サブリーダー（1名）：課題分析と企画の品質チェック
- 実行マネジャー（1名）：スケジュール調整と資料作成
- 記録係（1名）：議論内容の記録とレポート作成
- サポート（複数名）：各タスクの補助

→ 「役割が明確」なので、誰がどう貢献するかが見える

【グループ内の意思決定プロセス】
1. 「課題分析」段階
   - 全員で「課題について議論」（多角的な視点の収集）
   - リーダーが「複数の課題候補」を整理
   - 投票か、メリット・デメリット分析で「どの課題に取り組むか」を決定

2. 「企画提案」段階
   - リーダーが「複数の企画案」を作成
   - グループで「どの案が最適か」を議論
   - 「実行マネジャー」が「実現可能性」を検証
   - 投票で「最終案」を決定

3. 「実施」段階
   - 「実行マネジャー」が「週間スケジュール」を管理
   - 毎週「進捗確認ミーティング」を実施
   - 「想定外の課題」が生じた場合、「全員で対応案」を協議

【グループ内の対立が生じた場合】
- 津田先生が「調停者」としての面談を実施
- 目的：「対立を解決すること」ではなく、「複数の視点の価値」を理解させる
- 例：「リーダーはAの案を推しているが、サブリーダーはBの案を推している。両方の立場から『メリット』を出してみましょう」

【グループの「心理的安全性」の確保】
- 津田先生が「失敗を許容する」メッセージを繰り返す
- 「企画が却下されたら、その理由を『学習機会』として捉えよう」
- グループ内での「責任押し付け」を見かけたら、即座に介入
```

**効果：**
- グループ内の「役割と貢献」が明確
- 対立時の「解決方法」が用意されている
- 「失敗時の心理的負担」が軽減される
- ピアラーニングが本当に実現

### 提案3：キャリア形成での価値を「可視化」する
```
【修正案】学生が「何を得るか」を明確にする

【参加前に「期待値設定」】

ワークショップ：「このプロジェクトで、あなたは何を得たいのか？」

例：
- 学生A：「企画立案のスキルを身につけたい。就活で話題にしたい。」
- 学生B：「社会課題に対する『自分たちにできる貢献』を実感したい。」
- 学生C：「難病患者支援の世界に興味があるので、業界を理解したい。」

→ 各自が「個人的な目標」を設定

【習得すべきスキルの明示】

| スキル | 習得の機会 | 津田先生の支援 |
|---|---|---|
| 課題分析力 | 8月イベント後の「ワークショップ」 | 複数の分析フレームを教授 |
| 企画力 | 企画提案書の作成 | テンプレートとフィードバック |
| 実装力 | 9月～の企画実施 | 「困ったときの相談」体制 |
| 対人スキル | ステークホルダーとの協議 | 「事前打合せ」での練習 |
| メタ認知 | 月1回の「振り返りミーティング」 | 「なぜそう判断したのか」を問い直す |

【就活での「話題化」の方法】

津田先生が、プロジェクト終了後に「経歴書」を学生に発行

例：
```
「**社会課題解決プロジェクト」参加**
期間：2024年8月～12月
テーマ：難病患者支援の社会課題解決
成果：
- 8月のイベント参加で、難病患者のニーズを理解
- 学生グループで「啓発キャンペーン」を企画・提案
- 企業による承認を得て、実装を実施
- 実装結果：社内の難病への理解度が30%向上
担当教員：津田先生（連絡先）

【面接での説明例】
「このプロジェクトでは、『課題を発見してから、企画し、実装する』というフルサイクルを経験しました。
特に、複数のステークホルダー（企業、大学、NPO、患者さん）の関係性を理解し、
『どうやったら、すべての立場を尊重した企画ができるか』を考える力が付きました。」
```

【業界理解での付加価値】

企画実施段階で「業界人による『出張講座』」を実施

例：
- NPOのコーディネーター（加藤さん）から：「NPO業界の現状と課題」
- 企業CSR担当者（宮本さん）から：「企業の社会貢献活動」
- 難病患者団体の代表から：「患者さんの本当のニーズ」

→ 学生が「複数の立場の価値観」を学べる
```

**効果：**
- 学生が「自分の成長」を言語化できる
- 就活での「話題」として使用可能
- キャリア形成への「直結性」が見える

### 提案4：テーマ選択と決定権の確保
```
【修正案】学生に「選択と判断」の自由度を与える

【テーマ選択の自由度】

案1：「複数のテーマから選ぶ」
- あらかじめ「3～4つの社会課題候補」を企業と大学で用意
- 8月イベント参加後、学生グループが「どのテーマに取り組むか」を投票で決定
- 例：
  - テーマA：難病への偏見と情報発信
  - テーマB：障害者雇用の支援
  - テーマC：高齢者向けの社会参画プログラム

案2：「イベント参加後に、テーマ提案を募る」
- イベント参加で「複数の課題に気づく」
- 学生グループが「われわれが解決したい課題は◎◎」と提案
- 企業・大学で「提案の実現可能性」を検証
- 承認されれば、その課題に取り組む

→ 案2の方が「学生の主体性」が高い

【決定権の配分】

段階ごとの「誰が決めるか」を明示：

1. 「課題選択」：学生が決定（企業・大学は「実現可能性の制約」のみ指摘）
2. 「企画案の方向性」：学生が決定（企業・大学は「修正意見」を提示）
3. 「実装方法」：学生が決定（困ったときに企業・大学が「支援」）

→ 「企業に却下される」というリスクを軽減しながらも、学生の決定権を尊重

【「却下」されるリスクへの対応】

もし企画が「却下」されたら：

ステップ1：「却下理由」を学生に詳しく説明
- 「何がダメだったのか」を複数の視点から説明
- 例：「教育的には良いが、企業の資源では実現不可能」など

ステップ2：「修正の可能性」を検討
- 「この修正なら実現できるか」を協議
- 学生側：「修正に応じたくない」と言える権利も認める

ステップ3：「代替案」を用意
- 「別のテーマで、新たに企画提案を作成するか」を提案
- または「このプロジェクトを終了し、別のプロジェクトに参加するか」を提案

→ 「却下 = プロジェクト終了」ではなく、「学習機会」として位置づけ
```

**効果：**
- 学生が「やらされている感」を感じない
- 「失敗を恐れず、チャレンジする」姿勢が育つ
- 企業との「対等な関係」が形成される

### 提案5：成功体験と失敗からの学習を組み込む
```
【修正案】段階的な「達成感」と「学習サイクル」

【小さな成功体験の設計】

段階1：8月イベント参加
- 成功体験：「難病患者の声を直接聞けた」「複数の課題が見えた」
- 津田先生の支援：イベント参加直後に「ふりかえりシート」を記入
- 内容：「学んだこと」「驚いたこと」「疑問に思ったこと」

段階2：企画案の作成
- 成功体験：「複数の視点を『企画案』という形にまとめられた」
- 津田先生の支援：中間レビュー（8月24日）で「ここはいいね」と具体的にフィードバック
- 内容：「複数の視点を考慮している」「実現可能性をチェックしている」など

段階3：企画案の承認
- 成功体験：「企業と大学から『〇〇のような企画が必要だった』と言われた」
- 津田先生の支援：承認時に「あなたたちの企画が、実際に社会で実装される」ことを強調

段階4：企画実施
- 成功体験：月ごとに「小さな成果」を確認
- 津田先生の支援：月1回の「進捗確認ミーティング」で「ここはうまくいった」と指摘

段階5：最終的な成果確認
- 成功体験：「社員の意識変化」「患者さんからの反応」など、「目に見える成果」
- 津田先生の支援：「成果報告会」を実施し、学生が「自分たちの貢献」を実感

【失敗からの学習サイクル】

もし「企画が却下」される場合：

ステップ1：「感情の受け止め」
- 津田先生が個別に学生と面談
- 目的：「がっかり」「悔しい」という感情を否定しない
- メッセージ：「その気持ちは、とても大切です。それは『真剣に取り組んだ証』です」

ステップ2：「なぜ却下されたのか」を分析
- グループで「却下理由」を分析
- 「自分たちの判断のどこが『現実と乖離していたか』」を言語化
- 例：「企業の資源制約を甘く見ていた」「患者さんのニーズ理解が不足していた」

ステップ3：「学びを言語化」
- 「この失敗から、何を学んだか」をレポートにまとめる
- 例：「複数のステークホルダーのニーズを並立させることの難しさ」を学んだ

ステップ4：「次への挑戦」
- 「修正して、再提案するか」「別のテーマで新たに企画するか」を学生が選択
- 津田先生：「どちらの選択でも『学習としての価値』はある」と伝える

ステップ5：「最終評価」
- 失敗の有無ではなく「失敗からどう学んだか」で評価
- 成績評定：「企画の質」ではなく「学習プロセス」を評価

【振り返りの仕組み】

月1回の「振り返りミーティング」

進行者：津田先生 + 加藤さん（メンター）
参加者：学生グループ + 企業担当者（オプション）

内容：
- 「この月、何をしたか」を整理
- 「予定通りにいったか、困ったことはないか」を確認
- 「自分たちの判断は、後で考えると正しかったか」を問い直す
- 「次の月に向けて、何を改善するか」を決定

→ 「実行と学習」が統合される

【最終的な「学び」の確認】

プロジェクト終了後に「学び振り返りレポート」を提出

内容：
- 「このプロジェクトで、何を学んだか」（自分の言葉で）
- 「失敗や困難は、どのような学習機会だったか」
- 「この経験から、今後『社会に貢献する』ときに活かしたい学び」

→ メタ認知能力の発達を促す
```

**効果：**
- 小さな成功体験が積み重なり、「学習効果」が高まる
- 失敗が「学習の一部」として正当化される
- 学生が「自分たちの成長」を認識できる
