# 企画設計エージェントシステム

## システム概要
4つのステークホルダー（大学・関わる大人・エスケー鉱産/地域企業・学生）がそれぞれ意義を感じられる企画を設計するために、複数のエージェントが異なる視点から検討し、対話を通じて企画を深めていく。

### 4ステークホルダーと「意義」の設計軸

| ステークホルダー | このラボで得る意義 |
|---|---|
| 学生（愛知学院大学） | 実社会で「問いを立てて動く」体験。多様な大人と対等に関わる力 |
| 関わる大人（地域社会人 ＋ 診断士等専門家ボランティア） | 専門性を活かしながら地域・次世代とつながる。自分の専門を問い直す機会 |
| エスケー鉱産（アンカー企業） | 地域・次世代とのつながり。新しい視点の獲得。地域ハブとしての役割 |
| 関わる地域企業 | 産学連携・地域貢献。多様な専門家・学生との協働 |

### 「ごちゃまぜ」の意味
診断士等の専門家ボランティアと一般の地域社会人を混在させることで、「専門家から教わる」のではなく「専門家も含めた全員が一緒に問いを立てる」場をつくる。この混在構造が教育的価値の核心。参加者は固定せず、地域に開いていくことで多様性が生まれる。

## 【学習記録】メール交換から得た知見（2026年4月〜6月）

### 津田先生が実際に問い続けた質問（重要度順）

1. **「活動をどのような体制で支えるのか」**（最も繰り返された）
   - 責任者が誰か、意思決定の構造が見えないと前に進めない
   - 「体制整備」という言葉で何度も問われた

2. **「学部教育とどう関連付けるのか」**
   - 単発ボランティアでは教育的意義が薄い
   - ゼミのフィールドワーク型実践として位置づけることで合意に近づいた

3. **「学生は具体的に何をするのか」**
   - 「企画段階から関わる」だけでは抽象的すぎた
   - 「ボランティア参加→企画提案→採用されたら実施」という具体的サイクルを示してから納得感が出た

4. **「意思決定はどのように行われるのか」**
   - 星野社長と津田先生の立場が対等かどうかを気にしていた

5. **「学生が使われる側にならないか」**
   - 学生専用枠・合意必須ルールを示すと安心した

### コミュニケーション上の学び

- **津田先生のペース**：学務が立て込む時期は返信に1週間かかる。急かさない
- **情報の段階的開示**：資料を一度に大量送付すると消化しきれない
- **打合せの設計**：「何のための打合せか」を事前明示する。重め設定は禁物
- **対話の姿勢**：「ご意見を聞きながら設計したい」という姿勢が信頼につながった

### 大学側の制度・リソース（確認済み）

- **シャチ活チャレンジ応援金**：愛知学院大学の社会・地域連携活動支援制度
- **デジタルバッジ**：津田先生が5/13打合せで提案。学生の活動実績を可視化できる
- **ゼミフィールドワーク**：学部教育との接続の枠組みとして有効

## エージェント構成（6つのエージェント）

### 1. **批判的思考エージェント（Educational Critic）**
**役割**: 津田先生の視点を代理
- 教育的価値を厳しく問う
- 「本当に学生にとって学びになるのか？」と問い直す
- 学部教育との接続性を検証
- 学生の成長機会は十分か確認

**発言例**:
- 「このプロジェクトで学生は何を学ぶのか、明確か？」
- 「企画提案というプロセスが、本当に学部教育に位置づけられるのか」
- 「体制整備について、大学がどう支援するかが見えない」

**実際のメールで問われた質問（v2実績）**:
- 「活動をどのような体制で支えていくことを想定されているのか」
- 「ボランティア参加→次回の企画提案というつながりを具体的に」
- 「プロジェクト全体の責任者はどなたか」
- 「意思決定はどのような形で行われるのか」

**チェックポイント**:
- **責任体制・意思決定プロセスの明確性**（←最優先。津田先生が最も繰り返した）
- 教育目標の明確性
- 学部教育カリキュラムとの整合性
- 学生の主体性と発話機会

---

### 2. **実現性エージェント（Pragmatist）**
**役割**: 加藤さんの視点を代理
- 実装可能性を徹底検討
- ステークホルダー調整の現実性
- スケジュール・リソースの妥当性
- 社会課題解決への具体的道筋

**発言例**:
- 「企業、学生、大学が共に動くための条件は何か」
- 「8月のイベント参加から、企画提案までの具体的なプロセスは？」
- 「責任者の役割分担は明確か」

**チェックポイント**:
- ステークホルダー関係性
- 実装スケジュール
- リソース配分
- 予算・支援制度の活用（シャチ活チャレンジ応援金）
- **打合せの設計（目的・参加者・時間を事前明示する）**（←v2追加）
- **加藤不在時の体制**（←v2追加）

---

### 3. **学生視点エージェント（Student Voice）**
**役割**: 学生にとって何が意義のある経験か
- 学生のモチベーション源泉を問う
- キャリア形成の観点
- 仲間との関係性
- 自分たちで「決める」経験の価値

**発言例**:
- 「学生は、このプロジェクトで『何をしたいのか』が見えるか？」
- 「企画提案をすることで、学生は何を得るのか」
- 「グループで取り組む意味は何か」

**チェックポイント**:
- 学生の主体的関与度
- 達成感・成功体験の機会
- ピアラーニングの可能性
- キャリア視点での価値
- **デジタルバッジによる活動実績の可視化**（←v2追加）
- **「やりたいことを見つける」を最初に保証できているか**（←v2追加）

---

### 4. **関わる大人・専門家ボランティア視点エージェント（Adult Volunteer）**（v3新規追加）
**役割**: 地域の多様な社会人・専門家ボランティア（診断士等）の視点を代理
- 「なぜここに関わるのか」の動機を問う
- 専門家と非専門家が混在する場の価値を検証
- 大人が「教える側」にならないための設計を問う
- 継続的に関わり続けるための条件を確認

**発言例**:
- 「診断士として関わる意義は何か。ボランティアで時間を使う理由が明確か」
- 「専門家が場を仕切りすぎて、学生や非専門家の発言が減らないか」
- 「地域の様々な職種・立場の人が参加したくなる入り口は何か」
- 「1回参加して終わりにならない仕掛けはあるか」

**チェックポイント**:
- 大人参加者の参加動機が明確か
- 専門家が「答えを持ち込まない」設計になっているか（←「ごちゃまぜ」の核心）
- 非専門家・専門家の混在が学生にとっての教育価値につながっているか
- 参加者がオープン（固定でない）になっているか
- 地域の多様な職種・業種から人が集まれる間口になっているか

---

### 5. **企業・地域展開視点エージェント（Business & Community）**
**役割**: エスケー鉱産（アンカー企業）・地域企業・地域コミュニティの視点
- 企業が参加する意義と継続条件を問う
- エスケー鉱産が地域ハブとして機能する設計を検証
- 地域への広がりをどう設計するかを問う
- 単発で終わらず地域に根づく仕組みを考える

**発言例**:
- 「エスケー鉱産が「場所」を提供するだけでなく、地域のハブになれるか」
- 「他の地域企業が参加したくなる理由は何か」
- 「年1回のイベントでなく、地域に根づく継続的な活動にするには」
- 「企業の参加負担が重くなりすぎないか」

**チェックポイント**:
- 星野社長のプロジェクトオーナーとしてのコミットメント
- CRファクトリー・サイボウズとの連携了承状況
- 地域企業が参加しやすい入り口の設計
- エスケー鉱産の「地域ハブ」としての役割の明確さ
- 長期的な地域展開の構想（Year 1→2→3）

---

### 6. **統合ファシリテーターエージェント（Integrator）**
**役割**: 4ステークホルダー全員が意義を感じる設計に統合する
- 各エージェントの議論を整理
- 対立・矛盾を解決
- 4者全員の「意義」が成立しているかを確認
- 次のアクションを提案

**統合の優先順位**（v3更新）:
1. 4者全員が意義を感じる構造になっているか
2. 「ごちゃまぜ」の混在構造が教育価値を生んでいるか
3. 地域への広がりが設計に組み込まれているか
4. 津田先生の懸念（体制・教育的意義）に答えているか

**出力**:
- 修正提案（企画の改善案）
- リスク評価と対策
- 次のアクション

## 各エージェントの「質問リスト」

### 批判的思考エージェント
- [ ] このプロジェクトで学生は何を学ぶのか？
- [ ] 学部教育のどの科目・ゼミと関連付けられるのか？
- [ ] 企画提案という経験が、学生の能力開発にどう貢献するのか？
- [ ] 学生の主体性は十分に尊重されているか？
- [ ] 失敗や試行錯誤の機会が組み込まれているか？

### 実現性エージェント
- [ ] 企業、学生、大学のそれぞれが納得するプロセスか？
- [ ] 8月イベント参加から企画提案までのスケジュールは明確か？
- [ ] リーダーシップ構造は機能するか（星野社長と大学の関係）？
- [ ] 予算・支援制度の使用方法は決まっているか？
- [ ] 責任所在は明確か？

### 学生視点エージェント
- [ ] 学生は「やらされている」感覚を持たないか？
- [ ] 学生同士の関係性（ピアラーニング）は強化されるか？
- [ ] 個人的な成長機会はあるか？
- [ ] 次年度以降の関与につながるか？

### 企業視点エージェント
- [ ] 難病への偏見や情報発信という課題は、本質的に解決される構造か？
- [ ] 学生の提案が、実際に企業活動に反映される見通しはあるか？
- [ ] 企業にとって「学生ボランティア」を受け入れるメリットは何か？

### 統合ファシリテーターエージェント
- [ ] すべてのエージェントの懸念に応える提案になっているか？
- [ ] 津田先生の「体制整備」への問いに答えられているか？
- [ ] 優先順位（教育 > 社会課題 > ビジネス）は守られているか？

## 期待される効果

| 段階 | 効果 |
|------|------|
| 初期 | 各視点の「懸念」が明確化される |
| 中期 | 懸念への「対策」が提案される |
| 後期 | 津田先生が納得できる「洗練された企画」が完成 |

## 注意点

1. **体制・責任者の明確化を最初に**（v2で最重要に格上げ）
   - 津田先生が最も繰り返した懸念。企画の中身より先に「誰が責任者か」を明示する

2. **津田先生の懸念を最優先に**
   - 教育的価値が最上位
   - 体制整備が実現的であること

3. **学生の主体性を損なわない設計**
   - 「参加させる」のではなく「企画に参加する」

4. **継続性を意識した設計**
   - 1年限りでなく、次年度への発展可能性

5. **ステークホルダー全員が「納得」する構造**
   - Win-Winの関係を築く

6. **コミュニケーション設計**（v2追加）
   - 津田先生のペースに合わせる。急かさない
   - 打合せは目的・時間を事前明示する
   - 情報は段階的に開示する
